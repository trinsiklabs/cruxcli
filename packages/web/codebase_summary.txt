## PART 3: EXTERNAL APIS & INTEGRATIONS

### Quip API (Quip Documents)
  - Endpoint: https://platform.quip.com/1/
  - Used by: memory/import_chat.py, memory/source_chunker.py
  - Data: 4 Quip document collections (Belle, Chloe, Rebecca, Stephanie)
  - Auth: Bearer token in config.json["api_keys"]["quip"]

### Limitless AI API (Memory/note sync)
  - Endpoint: https://api.limitless.ai/v1
  - Used by: app/chat_importers/limitless.py
  - Auth: Bearer token in config.json["api_keys"]["limitless"]
  - Purpose: Import Limitless recordings and transcripts

### YouTube Data API v3
  - Used by: relationship_skills_app/knowledge_base/ modules
  - Auth: API key in config.json["api_keys"]["youtube_data_v3"]
  - Purpose: Research executor fetches video/channel metadata

### HTTP Requests (Low-level)
  - Used by: app/importers/bdsmlr.py, app/review_analyze.py
  - Purpose: Scraping blog content, downloading metadata

### NO Direct Anthropic/Claude API
  - All Claude interactions are via Claude Code (no SDK calls in Python)
  - Memory pipeline prompts are generated as text files, then fed to Claude

## PART 6: DEPENDENCIES (No Heavy External Dependencies)

Python stdlib only for core modules:
  - json, sqlite3, pathlib, datetime, collections, etc.

Optional/ML dependencies (marked as excluded from coverage):
  - torch (for embeddings.py)
  - clip (for clip_cache.py, clip_scorer.py, clip_db.py)
  - spacy (for NLP in analysis modules)
  - anthropic (NOT USED - Claude interactions via API only)
  - openai (NOT USED)

Lightweight dependencies actually used:
  - flask (web server, can be replaced with Phoenix)
  - requests (HTTP, minimal use)
  - dataclasses (standard library)

Implication: Minimal external library migration needed. Main work is:
  1. Flask → Phoenix/Elixir web framework
  2. sqlite3 → Ecto/PostgreSQL
  3. Python analysis pipeline stays Python (keep as workers) OR port to Elixir
